Archi TDD Based

Une partie de notre logique se situe dans les vues

Pour les tester facilement on a besoin de pouvoir leur injecter au minimum:

  • les templates
  • les models

this.recapContratCibleView = new LG.arbitrage.RecapContratCibleView({
    model: this.options.models.contrat,
    template: this.options.templates.$recapContratTemplate
});

Le routeur appelle une vue, qui contient une vue, qui contient une vue ...

On injecte depuis le haut de la pile les objets, collections, models, nécéssaires à toutes les vues

Chaque element peut donc être facilement mocké: tests plus précis

En prod on a donc besoin d'une partie de notre code qui lie,
DOM <-> templates <-> vues,
JSON <-> Models

Limites

  • pas d'IOC (dependency injection)
  • pas de factorisation entre les routes
  • pas de double binding automatique
  • trop de liberté ? C'est en faisant n'importe quoi...

Les models zombies

Késkispass

Lors de la construction de la vue Details, Backbone bind l'event change sur l'élément input contenu dans son el (http://backbonejs.org/#View-delegateEvents)


$('#zombieDetails').delegate('input', 'change', callbackQuiModifieLeModel);

#zombieDetails n'est jamais détruit, les events ne sont jamais debindé -> à chaque change, avalanche de callbacks
